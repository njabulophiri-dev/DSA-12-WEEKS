# 12-Week Technical Interview Prep Roadmap

A structured, self-paced plan to go from Java/DSA fundamentals to interview-ready in 12 weeks built for someone preparing for software engineering technical assessments.

## Goal

Not just to pass a coding test but to build genuine engineering depth: understanding *why* a data structure works the way it does, writing clean and tested code and being able to explain your reasoning clearly under pressure.

## Structure

The roadmap is split into four phases, each building on the last.

| Phase | Weeks | Focus |
|---|---|---|
| 1. Foundations | 1–3 | Arrays, ArrayList, Big O notation |
| 2. Key Structures | 4–6 | HashMap, HashSet, Stack, Queue |
| 3. Core Algorithms | 7–9 | Binary search, two-pointer technique, recursion |
| 4. Interview Simulation | 10–12 | Timed drills, edge-case testing, full mock assessment |

Time commitment: roughly 8–10 hours per week.

---

## Phase 1: Foundations (Weeks 1–3)

**Goal:** Get comfortable with control flow logic and Java's basic memory containers.

- **Week 1 — Arrays & Loops:** Traverse arrays with `for`, `while` and for-each. Build a min/max finder.
- **Week 2 — ArrayList:** Understand dynamic resizing behind the scenes. Practice `.add()`, `.get()`, `.remove()`, `.size()`. Build a string-reversal function.
- **Week 3 — Big O Notation:** Learn O(1) vs O(N) vs O(N²) in plain terms. Learn to spot nested loops that create quadratic time.

## Phase 2: Key Structures (Weeks 4–6)

**Goal:** Master the structures that turn slow code into fast code.

- **Week 4 — HashMap & HashSet:** Learn key-value pairs and why lookups are O(1). Build a word-frequency counter.
- **Week 5 — Stacks & Queues:** LIFO vs FIFO. Use a stack to validate balanced parentheses.
- **Week 6 — Review:** Solve 10 "Easy" problems on a coding practice platform using only arrays, ArrayLists and HashMaps. Prioritize clean variable names and comments.

## Phase 3: Core Algorithms (Weeks 7–9)

**Goal:** Learn the classic patterns interviewers test for.

- **Week 7 — Binary Search:** Understand why O(log N) beats linear search. Implement it with loops and pointers.
- **Week 8 — Two-Pointer Technique:** Use start/end index pointers to solve problems efficiently, e.g. finding a pair that sums to a target.
- **Week 9 — Recursion & Sorting:** Understand base cases and recursive calls. Study how Bubble Sort works conceptually and why built-in sort methods are more optimized.

## Phase 4: Interview Simulation (Weeks 10–12)

**Goal:** Train under time pressure and polish for the real thing.

- **Week 10 — Timed Drills:** Solve 2 problems daily under a 45-minute limit, focused on strings and arrays.
- **Week 11 — Edge-Case Engineering:** Break your own code before submitting. Always test with empty inputs, single-element arrays and negative numbers.
- **Week 12 — Full Mock Assessment:** Set aside a quiet 2-hour block. Attempt a full timed test without looking up answers.

| Session Type | Allocation | Activities |
|---|---|---|
| Theory & Visuals | 2 hours | Watch or sketch how each data structure executes step by step |
| Active Coding | 4 hours | Write Java syntax by hand, compile, debug |
| Problem Solving | 3 hours | Tackle timed practice problems on a coding platform |

---

## Habits That Separate Strong Candidates

These go beyond the syllabus but make the biggest difference in actual interviews:

- **Comment your complexity.** Annotate methods with time and space complexity: don't just solve, explain.
- **Build it from scratch once.** Implement a basic HashMap or Stack yourself to truly understand the mechanism underneath the built-in class.
- **Write tests, not just solutions.** Even a few JUnit assertions per problem shows engineering maturity most candidates skip.
- **Explain it out loud.** If you can't teach a concept simply, you don't understand it yet.
- **Document your progress.** A clean GitHub history and short READMEs on practice projects tell a stronger story than a perfect score alone.

---

## Tracking Progress

Use this checklist each week:

- [ ] Wrote code by hand; no copy-pasting solutions
- [ ] Can explain the week's core concept without notes
- [ ] Annotated code with Big O complexity
- [ ] Tested with edge cases, not just the happy path
- [ ] Reflected briefly on what went wrong and what to fix next week

---

## Notes

This roadmap assumes prior basic familiarity with Java syntax. Adjust pacing as needed; consistency matters more than speed.
